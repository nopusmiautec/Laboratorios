Preguntas y Respuestas - Laboratorio 1

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Respuesta: Internet es la red global de computadoras interconectadas, mientras que la World Wide Web (WWW) es un servicio que funciona sobre Internet para acceder a páginas web mediante HTTP.

2. ¿Cuáles son las partes de un URL?

Respuesta: Un URL se compone de:

  • Protocolo (ej: https://)
  • Dominio (ej: tec.mx)
  • Ruta (ej: /carpeta/Lab01.html)
  • Parámetros (opcional, ej: ?id=123)

3. Métodos HTTP

Ver métodos
  • GET: Solicita datos (ej: cargar una página).
  • POST: Envia datos (ej: formularios de login).
  • PUT/PATCH: Actualiza recursos.
  • DELETE: Elimina recursos.
  • HEAD: Solicita encabezados sin cuerpo.

4. ¿Qué método HTTP se usa en formularios con datos sensibles?

Respuesta: POST, porque los datos no se muestran en la URL (a diferencia de GET).

5. ¿Qué método HTTP se utiliza cuando se accede a una página a través de un URL?

Respuesta:

Cuando escribes una URL en tu navegador (ej: https://www.ejemplo.com), el navegador envía una solicitud GET al servidor. Este método:

  • Solo recupera datos (no los modifica).
  • Los parámetros (si existen) se ven en la URL (ej: ?busqueda=html).
  • Es cacheable y almacenable en el historial del navegador.

Ejemplo en código:

GET /index.html HTTP/1.1 Host: www.ejemplo.com

6. Códigos de estado HTTP

  • 200: Éxito (no es un error).
  • 404: No encontrado (error del cliente).
  • 500: Error interno del servidor (debe corregirlo el desarrollador).

11. Ciclo de vida vs. Ciclo de desarrollo de sistemas

Ciclo de vida del sistema de información (SLC)

Abarca todo el tiempo de existencia del sistema, incluso después de su desarrollo:

  1. Planificación: Definición de objetivos y viabilidad.
  2. Análisis: Requerimientos del negocio (no solo técnicos).
  3. Diseño: Arquitectura global (hardware, red, seguridad).
  4. Implementación: Instalación y despliegue.
  5. Mantenimiento: Soporte y actualizaciones.
  6. Disposición: Retiro o migración a nuevo sistema.
Ciclo de desarrollo de software (SDLC)

Fases técnicas para construir el software:

  1. Requerimientos: Especificaciones funcionales y técnicas.
  2. Diseño: Diagramas UML, prototipos.
  3. Codificación: Desarrollo del código.
  4. Pruebas: Unitarias, integración, rendimiento.
  5. Despliegue: Instalación en producción.
  6. Mantenimiento: Corrección de bugs y mejoras.

Modelos comunes: Cascada, Ágil (Scrum), Espiral.

Relación: El SDLC es una parte del SLC (enfocada solo en el software).

Preguntas y Respuestas - Laboratorio 3

1. Uso de !important en CSS

Recomendación: Evitar su uso excepto en casos extremos donde sea necesario anular estilos de terceros. Su abuso dificulta el mantenimiento del código.

2. Imágenes de fondo

Consideraciones: Deben optimizarse para rendimiento, tener contraste adecuado con el texto, y siempre incluir un color de fallback.

3. Unidades de medida (%, px, pt)

Recomendación: Usar % para diseños fluidos, px para medidas precisas, y evitar pt (está pensado para medios impresos). Preferir rem para tipografía.

4. CSS minimizado

Ventajas: Reduce tamaño de archivo (mejor rendimiento), disminuye ancho de banda, y acelera el parsing del navegador.

Preguntas y Respuestas - Laboratorio 4

1. ¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Diferencias:

  • Tipo de lenguaje: Java es compilado y fuertemente tipado, JavaScript es interpretado y débilmente tipado.
  • Uso principal: Java para aplicaciones empresariales/Android, JavaScript para web (frontend/backend con Node.js).
  • Ejecución: Java necesita JVM, JavaScript se ejecuta en navegadores o motores como V8.

Semejanzas:

  • Sintaxis similar (condicionales, bucles, operadores).
  • Ambos son orientados a objetos (aunque JavaScript usa prototipos).
  • Soportan funciones como ciudadanos de primera clase.

2. ¿Qué métodos tiene el objeto Date? (Menciona al menos 5)

  • getDate(): Devuelve el día del mes (1-31).
  • getFullYear(): Obtiene el año (4 dígitos).
  • getMonth(): Retorna el mes (0-11).
  • getHours(): Devuelve la hora (0-23).
  • toISOString(): Convierte la fecha a string en formato ISO.

3. ¿Qué métodos tienen los arreglos? (Menciona al menos 5)

  • push(): Añade elementos al final.
  • pop(): Elimina el último elemento.
  • map(): Crea un nuevo array aplicando una función a cada elemento.
  • filter(): Filtra elementos que cumplan una condición.
  • join(): Convierte el array a string uniendo elementos con un separador.

4. ¿Cómo se declara una variable con alcance local dentro de una función?

Usando let o const dentro del cuerpo de la función:

function ejemplo() {
                let local = "Solo visible aquí";
                const PI = 3.1416;
                }

Nota: var también declara variables locales, pero su alcance es la función completa (no el bloque).

5. ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

  • Contaminación del espacio global: Puede haber colisión de nombres con otros scripts.
  • Dificultad para depurar: Es más difícil rastrear cambios inesperados.
  • Acoplamiento innecesario: Las funciones dependen de estado externo, reduciendo reusabilidad.
  • Seguridad: Cualquier script puede modificar variables globales.

Alternativa: Usar módulos o closures para encapsular variables.

Sistema de Progresión en Videojuegos (Script 6)

Problemática: En muchos videojuegos, se necesita un sistema para gestionar el progreso del jugador (puntuación y nivel) de manera organizada y escalable.

Solución implementada:

  • Clase Jugador con propiedades: nombre, puntuacion, y nivel.
  • Método sumarPuntos(): Aumenta la puntuación y verifica si el jugador sube de nivel (cada 100 puntos × nivel actual).
  • Método subirNivel(): Incrementa el nivel automáticamente.

Ejemplo de uso:

const jugador = new Jugador("Ana");
        jugador.sumarPuntos(150); // Puntuación: 150, Nivel: 2

Contexto real: Sistemas similares se usan en juegos RPG (como Pokémon o Skyrim) para recompensar al jugador.
