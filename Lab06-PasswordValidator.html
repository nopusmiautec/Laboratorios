    Preguntas a responder - Laboratorio 06

    1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    Respuesta: Es una buena práctica porque permite una validación inmediata en el lado del cliente, mejorando la experiencia del usuario al proporcionar retroalimentación instantánea sin necesidad de recargar la página o esperar una respuesta del servidor. Esto reduce la carga en el servidor y evita enviar datos inválidos.

    2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    Respuesta: Las validaciones de JavaScript pueden saltarse deshabilitando JavaScript en el navegador, modificando el código mediante herramientas de desarrollo, o enviando directamente peticiones al servidor sin pasar por la interfaz de usuario. Por esta razón, las validaciones del lado del cliente nunca deben ser la única línea de defensa.

    3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    Respuesta: Aunque las validaciones de JavaScript pueden saltarse, siguen siendo una buena práctica porque proporcionan una mejor experiencia de usuario al detectar errores inmediatamente. Sin embargo, siempre deben complementarse con validaciones del lado del servidor para garantizar la seguridad e integridad de los datos. Las validaciones del cliente son para usabilidad, las del servidor son para seguridad.
