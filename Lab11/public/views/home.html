Descripción del archivo package.json

El archivo package.json es fundamental en cualquier proyecto Node.js y contiene:

1. Metadatos del proyecto

  • name: Nombre del proyecto
  • version: Versión actual del proyecto
  • description: Breve descripción del proyecto
  • author: Autor del proyecto
  • license: Tipo de licencia

2. Configuración principal

  • main: Punto de entrada de la aplicación (app.js)

3. Scripts

  • start: Inicia la aplicación con nodemon para desarrollo
  • test: Para ejecutar pruebas (aunque no tenemos configuradas)

4. Dependencias

  • dependencies: Paquetes necesarios para producción (express, body-parser)
  • devDependencies: Paquetes solo para desarrollo (nodemon)

Importancia

  • Permite gestionar fácilmente las dependencias del proyecto
  • Facilita la replicación del entorno de desarrollo
  • Contiene scripts útiles para automatizar tareas
  • Sirve como documentación del proyecto

Ejemplo de package.json:

{
  "name": "laboratorios",
  "version": "1.0.0",
  "description": "Aplicación web para laboratorios de Construcción de Software",
  "main": "app.js",
  "scripts": {
    "start": "nodemon app.js",
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "body-parser": "^1.20.2",
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.2"
  }
}
